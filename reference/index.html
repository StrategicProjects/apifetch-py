    Fetching

    Data fetching: single page and chunked retrieval.

    ApiError

    Bases: RuntimeError

    Raised when the API is unreachable or returns an HTTP error.

    fetch(api, name, limit=None, offset=0, query=None, verbosity=0, client=None)

    Fetch a single page from api and return it as a list of records.

    Parameters:

    Name Type Description Default
    api Api

    An :class:Api profile.

    		 required
    name str

    Token name to authenticate with (looked up via api.service).

    		 required
    limit Optional[float]

    Maximum records to request. None/inf means no limit.

    		 None
    offset int

    Starting record (omitted when 0).

    		 0
    query Optional[dict]

    Extra query-string filters.

    		 None
    verbosity int

    0 silent, >=1 progress messages.

    		 0
    client Optional[Client]

    Optional pre-built httpx.Client (useful for testing or connection reuse). One is created and closed per call otherwise.

    		 None

    Returns:

    Type Description
    list[dict[str, Any]]

    The parsed JSON body as a list of dictionaries.

    fetch_all(api, name, total_limit=None, chunk_size=50000, query=None, verbosity=0, client=None)

    Fetch every record by paging through api in chunks.

    Iteratively calls :func:fetch with an advancing offset until a chunk comes back empty or total_limit is reached. Columns listed in api.drop_cols are removed from each record.

    Fetch every record by paging through api in chunks.

    Iteratively calls :func:fetch with an advancing offset until a chunk comes back empty or total_limit is reached. Columns listed in api.drop_cols are removed from each record.

    API profile

    Describe an API endpoint together with its auth and pagination strategies.

    Parameters:

    Name Type Description Default
    endpoint str

    The base API URL.

    		 required
    service str

    Namespace used to look up the token (see :func:get_token).

    		 'apifetch'
    auth Auth

    An :class:Auth strategy. Defaults to bearer-token auth.

    		 AuthBearer()
    pagination Pagination

    A :class:Pagination strategy. Defaults to offset paging sent as HTTP headers.

    		 PaginateOffset()
    drop_cols tuple[str, ...]

    Keys to drop from each record after parsing (e.g. a status column).

    		 ()
    connect_hint str | None

    Optional extra line shown on a connection error (e.g. a VPN requirement).

    		 None

    Authentication strategies

    Bases: Auth

    Send the token verbatim in a header (default Authorization).

    This is what the Big Data PE API expects.

    Bases: Auth

    Send "<prefix><token>" in a header (default Authorization: Bearer).

    Bases: Auth

    Send the token in an arbitrary header (e.g. X-API-Key).

    Bases: Auth

    Send the token as a URL query parameter.

    Pagination strategies

    Bases: Pagination

    Send limit/offset as HTTP headers (default) or query parameters.

    Non-positive, None and infinite values are omitted.

    Bases: Pagination

    Send no pagination parameters.

    Token management

    Store token for name in an environment variable.

    Refuses to overwrite an existing, non-empty variable.

    Return the token stored for name/service, or None if missing.

    Remove the token stored for name/service if present.

    Return the names (without the service prefix) of stored tokens.
